# Code of Conduct

PatchWarden follows a simple maintainer standard: be respectful, specific, and
security-conscious.

## Expected Behavior

- Treat contributors and users with patience and professionalism.
- Keep reports actionable: include versions, operating system, reproduction
  steps, and redacted logs when useful.
- Respect local privacy. Do not post API keys, tokens, cookies, `.env` files,
  browser state, private task artifacts, or screenshots with private account
  details.
- Use GitHub private security advisories for suspected vulnerabilities or leaked
  credentials.

## Unacceptable Behavior

- Harassment, insults, threats, or discriminatory language.
- Posting another person's private data or credentials.
- Pressuring maintainers to run unsafe commands, bypass security guards, or
  publish unreviewed releases.
- Deliberately submitting misleading issue reports, fake adoption evidence, or
  fabricated security claims.

## Enforcement

The maintainer may edit, hide, or remove comments that violate this code of
conduct and may block users who repeatedly act in bad faith. Security reports
will be handled privately when they involve exploitable behavior or sensitive
data.
